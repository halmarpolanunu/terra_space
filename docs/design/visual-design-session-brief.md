# Terra Space Visual Design Session Brief

## Purpose of each page

- **Dashboard:** later shows approved-event summaries, map, timeline, and filters.
- **Documents:** later supports manual document drafts, selection, and batch processing.
- **Event Review:** later supports review and correction of AI-extracted event drafts.
- **Events:** later lists approved events and their details.
- **Settings:** later configures local LM Studio and editable event types.

## Confirmed constraints

- All interface text is English.
- The application is a local, desktop-first browser workspace for one person.
- Navigation must stay clear, keyboard accessible, and readable with text labels in addition to icons or color.
- The map must remain fully local and must not require a cloud map provider.
- Phase 1 uses a deliberately neutral shell; this is not final visual styling.

## Decisions reserved for the session

- Visual references and overall mood.
- Palette and contrast approach.
- Typography and information density.
- Light mode, dark mode, or both.
- Component language: panels, tables, controls, navigation, and empty states.
- Map colors, borders, labels, and interaction behavior.
- Motion and animation level.

No choices in this list have been made yet.
